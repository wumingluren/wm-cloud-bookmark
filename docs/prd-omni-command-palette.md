# 无名云书签 Omni 命令面板 PRD

## 1. 文档信息

- 项目名称：无名云书签 Omni
- 文档类型：产品需求文档（PRD）
- 当前版本：v0.1
- 文档日期：2026-04-01
- 参考产品：[alyssaxuu/omni](https://github.com/alyssaxuu/omni)
- 参考说明：参考产品公开信息显示，其核心是一个浏览器内全局命令面板，统一处理 `tabs / bookmarks / history / actions` 搜索与操作，并支持 `/tabs`、`/bookmarks`、`/history`、`/actions`、`/remove` 等命令前缀

## 2. 项目背景

无名云书签当前已经具备以下能力：

- 飞书云书签存储与搜索
- 新标签页推荐与搜索
- 右键收藏当前页面到飞书
- 全站内容脚本注入能力
- 基于 Shadow DOM 的浮层承载能力

当前产品的主要短板是：

- 搜索入口只存在于新标签页，不能在任意页面快速呼出
- 搜索范围偏窄，主要是飞书云书签与浏览器默认搜索
- 浏览器本地对象能力没有统一收口，包括标签页、历史记录、本地书签、快捷动作
- 用户需要在多个入口之间切换，缺少一个像 Omni 那样的全局操作入口

因此需要新增一个“全局命令面板”，把“找内容”和“做动作”统一到同一个输入框中。

## 3. 产品目标

### 3.1 总体目标

将无名云书签升级为“浏览器内统一命令入口”，允许用户在任意页面通过快捷键呼出面板，完成以下高频任务：

- 搜索飞书云书签
- 搜索浏览器本地书签
- 搜索当前打开标签页
- 搜索浏览历史
- 执行浏览器动作
- 调起默认搜索引擎搜索

### 3.2 成功标准

- 用户可在任意网页通过快捷键打开命令面板
- 用户可纯键盘完成搜索、选择、执行、关闭全流程
- 结果来源清晰，支持飞书云书签与浏览器本地对象统一检索
- 飞书服务异常时，本地 tabs/bookmarks/history/actions 仍可正常使用
- 面板打开与本地搜索响应足够快，达到“接近原生”的主观体验

### 3.3 核心差异化

相较于 Omni 原版，本项目的核心差异不是单纯复刻浏览器对象管理，而是新增“飞书云书签”这一跨设备云数据源，使用户在同一入口内完成“浏览器内对象 + 云端收藏”统一检索。

## 4. 用户画像与使用场景

### 4.1 用户画像

- 重度书签用户：收藏量大，需要快速命中历史积累
- 多设备办公用户：希望在公司电脑、家里电脑、移动端之间共享链接
- 浏览器重度标签页用户：需要快速切换、关闭、整理标签页
- 知识管理用户：使用飞书多维表格进行结构化整理

### 4.2 核心场景

- 在任意网页按快捷键，输入关键词，直接打开飞书中的某个收藏
- 在标签页很多时，输入域名或标题关键字，快速切换到对应标签页
- 在想不起是否收藏过某个站点时，统一搜索飞书和浏览器本地书签
- 在浏览网页时，快速把当前页收藏到飞书
- 输入命令前缀，只搜索某一类数据源，例如 `/tabs figma`
- 输入动作命令，快速执行关闭标签页、固定标签页、打开配置页等动作

## 5. 与现有项目的衔接

当前仓库已有可直接复用的基础能力：

- 新标签页搜索入口：`src/newtab/components/SearchBar.vue`
- 飞书推荐与搜索消息路由：`src/background/main.ts`
- 飞书书签搜索服务：`src/services/feishu.ts`
- 全站内容脚本注入与 Shadow DOM 容器：`src/contentScripts/index.ts`
- 内容脚本浮层容器：`src/contentScripts/views/App.vue`
- 右键收藏链路：`src/background/services/menuService.ts`

本次需求不应推翻现有新标签页，而应在现有扩展结构上新增一个“全局命令面板系统”，并与新标签页复用部分搜索 Provider 和动作能力。

## 7. 交互入口

### 7.1 唤起方式

- 支持浏览器快捷键唤起命令面板
- 建议默认快捷键：`Cmd/Ctrl + Shift + K`
- 支持从扩展图标点击进入面板
- 后续可支持侧边栏常驻模式

### 7.2 关闭方式

- `Esc` 关闭面板
- 点击遮罩层关闭
- 执行动作后按配置决定是否自动关闭

### 7.3 焦点行为

- 面板打开后输入框自动获取焦点
- 关闭后恢复到原页面焦点
- 组合输入法输入期间不得错误触发执行

## 8. 信息架构

### 8.1 一级搜索来源

- 飞书云书签
- 浏览器本地书签
- 当前标签页
- 浏览历史
- 动作命令
- 默认搜索引擎搜索候选

### 8.2 命令模式

支持以下命令前缀：

- `/tabs`：仅搜索当前打开标签页
- `/bookmarks`：仅搜索浏览器本地书签
- `/feishu` 或 `/cloud`：仅搜索飞书云书签
- `/history`：仅搜索浏览历史
- `/actions`：仅搜索动作命令
- `/remove`：进入删除模式

支持中文别名：

- `/标签页`
- `/书签`
- `/云书签`
- `/历史`
- `/动作`
- `/删除`

### 8.3 默认模式

无命令前缀时执行聚合搜索，对多数据源结果进行混排。

## 9. 用户流程

### 9.1 默认聚合搜索

1. 用户按快捷键打开面板
2. 输入关键词
3. 系统同时查询启用中的 Provider
4. 面板按统一规则排序并展示结果
5. 用户通过键盘选择一项
6. `Enter` 执行主动作

### 9.2 前缀定向搜索

1. 用户输入 `/tabs figma`
2. 系统解析到 `tabs` 模式
3. 仅调用 Tab Provider
4. 展示与标签页相关结果
5. 用户执行“切换标签页”或次级动作

### 9.3 动作执行

1. 用户输入 `/actions close current tab`
2. 系统只展示动作命令结果
3. 用户选中“关闭当前标签页”
4. 系统执行动作
5. 返回成功、失败或确认反馈

### 9.4 删除模式

1. 用户输入 `/remove`
2. 面板切换到危险操作模式
3. 用户继续搜索目标对象
4. 结果主动作变为“删除”
5. 系统执行删除前进行确认

## 10. 功能需求

### 10.1 全局命令面板

#### 需求说明

在任意网页中提供一个可快速唤起的搜索与动作面板。

#### 功能要求

- 使用内容脚本渲染浮层
- 使用 Shadow DOM 保证样式隔离
- 面板需支持遮罩层、容器、输入框、结果列表、状态栏
- 支持键盘导航：
  - `ArrowUp/ArrowDown` 切换选项
  - `Enter` 执行
  - `Esc` 关闭
  - `Tab` 或 `Shift+Tab` 切换模式/动作焦点
  - `Cmd/Ctrl+Enter` 执行次级动作（可选）
- 支持鼠标点击执行
- 支持空态、加载态、错误态

### 10.2 查询解析器

#### 需求说明

对用户输入进行语法解析，识别命令模式、关键词、危险操作模式。

#### 功能要求

- 识别命令前缀
- 识别是否为空查询
- 将查询拆分为：
  - 原始输入
  - 规范化输入
  - 模式
  - 实际查询关键字
  - 是否为危险模式
- 支持中英文命令别名
- 对非法命令给出纠错提示

### 10.3 Provider 聚合搜索

#### 需求说明

以统一接口对接多个数据源，并将结果聚合后返回给 UI。

#### 功能要求

- 定义统一 Provider 接口
- 每个 Provider 至少实现：
  - `search(query)`
  - `listRecent()`
  - `getPrimaryAction()`
  - `getSecondaryActions()`
- 聚合层按查询模式决定调用哪些 Provider
- 支持并发查询
- 支持结果归一化
- 支持排序与截断
- 支持 Provider 超时与失败隔离

### 10.4 飞书云书签 Provider

#### 需求说明

复用现有飞书搜索能力，作为 Omni 的核心差异化数据源。

#### 功能要求

- 支持搜索标题、URL、标签
- 支持展示来源配置名
- 支持展示标签
- 支持执行“打开链接”
- 支持执行“复制链接”
- 支持执行“收藏当前页到飞书”
- 支持多飞书配置模式：
  - 仅当前活动配置
  - 搜索全部配置
- 支持本地缓存轻量索引
- 支持远端失败时展示错误但不阻断其他结果

#### 数据字段要求

- 书签 ID
- 标题
- URL
- 域名
- 标签数组
- 来源配置 ID
- 来源配置名称
- 最近更新时间（若可获取）

### 10.5 浏览器本地书签 Provider

#### 需求说明

接入浏览器原生 bookmarks API。

#### 功能要求

- 支持按标题和 URL 搜索
- 支持展示目录路径
- 支持打开链接
- 支持删除书签
- 支持添加当前页面到浏览器书签
- 支持定位至书签目录信息

### 10.6 当前标签页 Provider

#### 需求说明

接入浏览器 tabs API，用于快速切换和管理标签页。

#### 功能要求

- 搜索所有打开标签页
- 支持按标题、URL、域名搜索
- 支持显示：
  - 标签页标题
  - URL 或域名
  - favicon
  - 所属窗口
  - 是否固定
  - 是否静音
- 支持动作：
  - 切换到标签页
  - 关闭标签页
  - 固定/取消固定
  - 静音/取消静音
  - 复制链接

### 10.7 浏览历史 Provider

#### 需求说明

接入浏览器 history API。

#### 功能要求

- 支持按标题、URL 搜索历史记录
- 支持展示最近访问时间
- 支持打开历史项
- 支持删除单条历史
- 支持显示“最近访问”默认结果

### 10.8 动作命令 Provider

#### 需求说明

将浏览器内高频操作包装成可搜索命令。

#### 本期动作清单

- 打开新标签页
- 打开扩展配置页
- 打开侧边栏
- 用默认搜索引擎搜索当前输入
- 收藏当前页到飞书
- 收藏当前页到浏览器书签
- 复制当前页标题和链接
- 关闭当前标签页
- 关闭其他标签页
- 关闭右侧标签页
- 固定/取消固定当前标签页
- 静音/取消静音当前标签页

#### 二期候选动作

- 打开最近关闭标签页
- 按域名批量关闭标签页
- 清理单站点历史
- 清理缓存和 Cookie
- 打开站点快捷动作模板

### 10.9 默认搜索引擎候选

#### 需求说明

当用户输入普通关键词时，除本地和云结果外，还需提供“使用默认搜索引擎搜索”的候选项。

#### 功能要求

- 始终在聚合结果中提供搜索引擎候选项
- 展示文案如“搜索网页：{query}”
- 执行动作为新标签页搜索

### 10.10 最近使用与个性化排序

#### 需求说明

记录用户的最近执行历史和动作偏好，提升命中效率。

#### 功能要求

- 记录最近执行的结果项
- 记录最近动作
- 记录最近搜索词
- 将最近使用作为空查询默认结果来源之一
- 为排序增加最近使用权重
- 支持在设置中清除最近记录

## 11. 结果展示规范

### 11.1 结果项通用字段

- 图标
- 标题
- 副标题
- 来源标记
- 主动作标签
- 可选次级动作入口
- 高亮命中关键字

### 11.2 分组展示

聚合搜索结果按以下优先顺序分组：

1. 最近使用
2. 飞书云书签
3. 当前标签页
4. 浏览器本地书签
5. 浏览历史
6. 动作命令
7. 搜索引擎候选

说明：

- 当某分组无结果时不展示分组标题
- 每组可配置显示上限
- 聚合模式下也可支持“完全混排”，但 UI 仍需保留来源标签

### 11.3 排序规则

建议综合以下因素计算排序分数：

- 标题前缀命中优先
- 完整词命中优先
- 标题命中高于 URL 命中
- 当前窗口标签页加权
- 最近访问历史加权
- 最近执行记录加权
- 飞书标签命中加权

## 12. 数据模型

建议定义统一结果结构：

```ts
type OmniResultType =
  | 'feishu-bookmark'
  | 'browser-bookmark'
  | 'tab'
  | 'history'
  | 'action'
  | 'search-suggestion'

interface OmniResult {
  id: string
  type: OmniResultType
  source: string
  title: string
  subtitle?: string
  url?: string
  icon?: string
  keywords?: string[]
  score: number
  payload: Record<string, unknown>
  primaryAction: OmniAction
  secondaryActions?: OmniAction[]
}

interface OmniAction {
  id: string
  label: string
  kind: 'open' | 'switch' | 'delete' | 'command' | 'copy'
  dangerous?: boolean
}
```

## 15. 性能要求

- 面板唤起时间目标：小于 150ms
- 本地 Provider 首屏结果目标：小于 100ms
- 飞书搜索应做 debounce，建议 150ms 到 250ms
- Provider 查询失败不得导致整个面板不可用
- 大量标签页情况下仍需保持键盘导航流畅

## 16. 可用性与无障碍

- 支持纯键盘操作
- 支持清晰焦点态
- 支持屏幕宽度较小时的自适应布局
- 支持中文和英文搜索
- 支持 IME 输入
- 危险操作需明确样式提示

## 17. 安全与隐私

- 不将浏览器历史、标签页、本地书签上传到飞书
- 不将用户查询词发送到第三方服务，除非用户显式执行搜索引擎搜索
- 危险操作必须有确认机制或可控开关
- 飞书凭证沿用现有存储机制，不新增外部服务依赖

## 20. MVP 验收标准

- 在任意网页可通过快捷键打开 Omni 面板
- 面板支持输入关键词并展示聚合结果
- 结果中至少包含：飞书云书签、标签页、本地书签、历史、动作、搜索引擎候选
- 支持 `/tabs`、`/bookmarks`、`/feishu`、`/history`、`/actions`
- 键盘上下选择与 `Enter` 执行工作正常
- 关闭标签页、切换标签页、收藏当前页到飞书动作可执行
- 飞书未配置时可清晰提示，但不影响其他本地功能
- 历史权限缺失或浏览器不支持时，可降级并提示

## 21. 风险与注意事项

- 飞书搜索为远端请求，响应速度不可控，必须做降级和缓存
- 浏览历史和删除类动作属于高风险操作，需额外确认
- Firefox 与 Chromium 在 `search`、`sidePanel`、快捷键行为上可能有差异
- 内容脚本浮层与宿主页面快捷键、焦点、输入法可能存在冲突，需要重点测试

## 22. 开发前置结论

从当前仓库结构看，本项目非常适合做这次复刻，原因如下：

- 已有浏览器扩展基础设施
- 已有全站内容脚本和 Shadow DOM 容器
- 已有飞书搜索服务
- 已有新标签页搜索 UI 可复用
- 已有后台消息处理和浏览器权限基础

真正需要新增的不是“能不能做”，而是把现有离散能力统一抽象成一套 Omni 架构。
